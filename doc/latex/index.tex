 {\ttfamily GRACE} is an evolution framework that uses Finite Difference methods to simulate relativistic spacetimes and plasmas.\hypertarget{index_autotoc_md24}{}\doxysection{Building the code}\label{index_autotoc_md24}
The build system used for this project is Cmake. {\ttfamily GRACE} depends on some external libraries, which should be provided at build times. These are\+: MPI, Kokkos, p4est, Catch2 (for testing only), and YAML-\/cpp. Inside {\ttfamily env/} are some example configuration files that set the required environment variables to enable Cmake to find all dependencies. {\ttfamily GRACE} supports 2 and 3 spatial dimensions. The number of spatial dimensions in {\ttfamily GRACE} is a compile-\/time switch that can be set via the {\ttfamily GRACE\+\_\+\+NSPACEDIM} cmake variable (see below for an example). The default is 2 dimensions. Moreover, {\ttfamily GRACE} leverages Kokkos as a shared-\/memory parallelization layer to achieve a (more or less) backend independent code structure. The available backends are\+: CUDA device, HIP device, host Open\+MP and host serial. The active backend is determined at build time through the {\ttfamily TUHNDER\+\_\+\+ENABLE\+\_\+\{DEV\}} variables, where {\ttfamily DEV} can be {\ttfamily CUDA}, {\ttfamily HIP}, {\ttfamily OMP}, or {\ttfamily SERIAL}. Note that the Kokkos library being linked must also have been compiled with this device, this is checked at build time. An example of how to build Thuder is\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{cmake -\/Bbuild -\/S./ -\/DGRACE\_NSPACEDIM=3 -\/DGRACE\_ENABLE\_CUDA=ON -\/DCMAKE\_BUILD\_TYPE=Debug}

\end{DoxyCode}
 Which sets up a {\ttfamily CUDA} backend, compiles the code in 3D, and includes all debug flags in the compilation.\hypertarget{index_autotoc_md25}{}\doxysection{Building the docs}\label{index_autotoc_md25}
